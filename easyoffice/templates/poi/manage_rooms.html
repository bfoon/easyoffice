{% extends "base.html" %} {% comment %} apps/poi/templates/poi/manage_rooms.html ───────────────────────────────────────── Staff-side admin page to manage which named staff are added to POI channels. This is the single, auditable exception to POI confinement: an admin/CEO may add a non-admin staffer to a channel that contains a POI. The page talks to: GET poi/manage/rooms/ → list POI channels + members GET poi/manage/staff-search/?q=&room_id= → search addable staff POST poi/manage/rooms//add-staff/ → add a staffer (records opt-in) POST poi/manage/rooms//remove-staff// → remove + revoke opt-in Access is gated server-side by AdminPOIManageMixin; a POI can never reach here. {% endcomment %} {% block title %}POI Channels{% endblock %} {% block content %}

POI Channels

Add or remove named staff from channels that include a person of interest.

Adding a staff member to a POI channel lets that staffer see and message the POI, and lets the POI see that staffer's name and messages. Only add people who are meant to have that visibility. Removing a staffer revokes their access to the channel immediately.
